## CRITICAL — Must fix before merge

### 1. `escalation.last_commentary` does not exist
**File:** `frontend/src/app/escalation/[id]/page.tsx:208`
```tsx
<KarenBossCard commentary={escalation.last_commentary ?? "ANALYZING TARGET VULNERABILITIES..."} />
```
`last_commentary` is not a field on the `Escalation` type (frontend or backend). This will fail TypeScript strict mode compilation.

**Fix:** Derive from the latest `commentary` event in the events array:
```tsx
const lastCommentary = [...events].reverse().find(e => e.type === "commentary");
const commentaryText = lastCommentary?.type === "commentary" ? lastCommentary.text : "ANALYZING TARGET VULNERABILITIES...";
```

### 2. `isPaymentDetected` defined but never used
**File:** `frontend/src/app/escalation/[id]/page.tsx:63`
```tsx
const isPaymentDetected = events.some((e) => e.type === "payment_detected");
```
Variable is computed but never rendered. The old payment banner UI was removed but no replacement was added. Payment detection is a core demo feature — Bharath pays via Venmo and the UI needs to show it.

**Fix:** Add a payment-detected banner similar to the response-detected one, with the "Initiate De-Escalation" button.

### 3. Deescalation UI completely removed
The PR deletes `DeescalationSequence.tsx` and `EscalationTimeline.tsx` without any replacement for deescalation step rendering. The new `EscalationTower.tsx` only shows the level indicator — it doesn't render `deescalation_step` events at all.

When the operator clicks "De-escalate", the backend emits 7-8 `deescalation_step` events. Currently **nothing renders them** in the PR.

**Fix:** Add deescalation step rendering — either in the sidebar, as a modal, or inline below the tower.

### 4. Font mismatch: layout.tsx not updated
The PR switches CSS to VT323/Silkscreen fonts (in `globals.css` and `pixel-theme.css`), but `layout.tsx` still imports and applies DM Mono/DM Sans/Syne via `next/font/google`. The old font CSS variables override the new ones.

**Fix:** Update `layout.tsx` to import VT323 and Silkscreen instead of DM_Mono/DM_Sans/Syne.
