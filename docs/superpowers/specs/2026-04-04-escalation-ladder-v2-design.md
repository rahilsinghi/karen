# Escalation Ladder v2 — Design Spec

**Goal:** Redesign Karen's 10-level escalation ladder so every level fires a unique channel with real integrations, no stubs, no repeats, and maximum demo impact.

**Replaces:** The original ladder (4x email, 2x SMS, 2 stubs, broken Twitter). This spec is authoritative — it supersedes the level definitions in CLAUDE.md.

---

## The New Ladder

| Level | Channel | What Karen Does | Demo Moment |
|-------|---------|-----------------|-------------|
| 1 | **Email** | Warm first contact from Karen's Gmail | Standard opener |
| 2 | **SMS** | Text message to target's phone | Phone buzzes on mirrored screen |
| 3 | **WhatsApp + Voice Call** | WhatsApp message + Karen calls (fire-and-forget) | Phone rings in the room |
| 4 | **OSINT Research + SMS** | Pre-cached discovery animation, then sends "I know where you work 🙂" | Audience goes quiet, then laughs |
| 5 | **Email CC** | Formal email CC'ing the "discovered coworker" (a circle member) | Payoff from the research step |
| 6 | **Slack** | Posts in #karen-escalations channel | Real Slack message visible to audience |
| 7 | **Discord @everyone** | Posts in demo server | Audience phones ping |
| 8 | **Google Calendar** | Creates event "Discuss $23 Debt" and invites target | Calendar notification on phone |
| 9 | **GitHub Open Matters** | Commits to public repo, page goes live | Permanent public record |
| 10 | **FedEx Letter** | Pre-generated legal letter + rate quote + QR code + PDF | Hold physical letter up on stage |

## Level-by-Level Spec

### Level 1: Email

**Channel:** `email`
**Integration:** Resend API (primary) or Karen's Gmail SMTP (fallback)
**What fires:** Single email to target's email address
**Message:** Generated by personality service. Subject + body. Warm, friendly, one emoji for passive-aggressive.
**Karen's note:** Internal monologue about first contact.
**Audio:** Quip on fire, commentary during countdown.

### Level 2: SMS

**Channel:** `sms`
**Integration:** Twilio SMS API
**What fires:** Text message to target's phone number
**Message:** Generated by personality service. Body only (no subject for SMS). Short, direct.
**Demo impact:** Bharath's mirrored phone buzzes visibly.
**Audio:** Quip, commentary during countdown.

### Level 3: WhatsApp + Voice Call

**Channel:** `whatsapp`, `voice_call`
**Integration:** Twilio WhatsApp API + Twilio Voice (Polly.Joanna TTS)

**What fires:** WhatsApp message first, then voice call initiated as fire-and-forget.

**Voice call — fire-and-forget:**
- The call is initiated but Karen does NOT wait for it to complete.
- At 5s/10s demo intervals, the call may still be connecting when Level 4 starts. This is intentional — the overlap layers nicely (OSINT animation starts while phone is still ringing).
- Karen's commentary acknowledges this: "I called. He may still be on hold. I did not wait."

**Voicemail branch (answeredBy handling):**
- If `answeredBy: human` → Karen reads the TTS message normally.
- If `answeredBy: machine` → Karen leaves the SAME message as a voicemail.
- Karen's commentary for voicemail: "He sent me to voicemail. I left one. It was thorough."
- TwiML must use `<Record>` or `machineDetection: DetectMessageEnd` to handle the voicemail beep timing.
- Fixed closing on all calls: "This was a message from Karen Automated Correspondence Systems. Goodbye."

**Demo impact:** Phone rings in the room. If Bharath ignores it (likely), Karen leaves a voicemail — even funnier.
**Audio:** Quip after fire, commentary during countdown.

### Level 4: OSINT Research + SMS Payoff (NEW)

**Channel:** `research`, `sms`
**Integration:** Pre-cached research results displayed via SSE events, then one SMS via Twilio.

**Phase 1 — Research animation:**
Karen "investigates" the target using pre-cached data, emitted as timed SSE events:

| Step | SSE Event | Delay After |
|------|-----------|-------------|
| 1 | "Scanning {target_name}'s public web presence..." | 0.4s |
| 2 | "Found portfolio at {portfolio_url}" | 0.4s |
| 3 | "Work history: {employer} — {role}" | 0.4s |
| 4 | "Domain discovered: {domain}. Inferring work email..." | **0.8s** (longer pause — the scary moment) |
| 5 | "Work email: {first}@{domain}. Confidence: high." | 0.4s |
| 6 | "Identifying colleagues for visibility escalation..." | 0.4s |
| 7 | `research_discovery` event with full payload | — |

**The work email moment (step 4→5):** This is the beat that makes the audience go quiet. The 0.8s pause before the email appears is intentional — it creates tension. The typewriter effect on the email address should be slightly slower than other steps.

**Phase 2 — SMS payoff:**
After the research animation completes, Karen sends ONE SMS to the target:
> "I know where you work 🙂"

This is NOT generated by the personality service — it is hardcoded. Same text every time.
Karen's commentary: "Just a friendly reminder that I am thorough."

**Pre-cached data structure:**
```python
@dataclass
class ResearchResult:
    target_name: str
    portfolio_url: str          # e.g. "rahilsinghi.com"
    employer: str               # e.g. "Kismet"
    role: str                   # e.g. "AI/Data Engineering Intern"
    domain: str                 # e.g. "makekismet.com"
    work_email: str             # e.g. "rahil@makekismet.com"
    coworker_name: str          # Actually a circle member
    coworker_email: str         # That circle member's real email
    discovery_steps: list[str]  # The narrative steps above
```

**Storage:** `backend/data/research_cache.json` — one entry per target member. Pre-populated before demo.

**Frontend:** New `ResearchAnimation` component in the timeline. Terminal-style green text (#22c55e) on dark background. Each step appears with typewriter effect. Work email step has a longer reveal.

**De-escalation:** Nothing to reverse for the research. The SMS cannot be unsent.

### Level 5: Email CC

**Channel:** `email_cc`
**Integration:** Resend API with CC field
**What fires:** Formal email to target, CC'ing the "discovered coworker" from Level 4's research results.
**Message:** Generated by personality service with `cc_name` context. Tone shift — now there's an audience.
**CC recipient:** The coworker from the research cache, which maps to a real circle member's email.
**De-escalation:** Apology sent to CC'd contact.

### Level 6: Slack (NEW)

**Channel:** `slack`
**Integration:** Slack Web API (`chat.postMessage`) via Bot token
**What fires:** Message posted to #karen-escalations channel in Karen's Slack workspace.
**Message:** Generated by personality service. Body only. Can include Slack mrkdwn (bold, links).
**Bot identity:** "Karen" bot with 🦞 emoji as avatar.
**Env vars:**
```
SLACK_BOT_TOKEN=xoxb-...
SLACK_CHANNEL_ID=C...
```
**De-escalation:** Delete message via `chat.delete` API using stored `ts` (message timestamp).
**Demo impact:** Audience can be invited to the Slack workspace (second QR code alongside Discord).

### Level 7: Discord @everyone

**Channel:** `discord`
**Integration:** Discord Bot API (existing, working)
**What fires:** @everyone message in #karen-text-demo-channel
**Message:** Generated by personality service. Includes @everyone mention.
**Demo impact:** Every audience member's phone pings. Signature demo moment.
**De-escalation:** Delete message via Discord API (existing, working).

### Level 8: Google Calendar (moved from L9)

**Channel:** `calendar`
**Integration:** Google Calendar API via Karen's Gmail account
**What fires:** Creates a calendar event titled "Discuss $23 Debt — Karen Automated Correspondence Systems" with:
  - Target as invited attendee (their email)
  - Event time: 30 minutes from now
  - Description: Generated by personality service
  - 3 reminders: 10min, 5min, 1min before
  - Location: "Wherever accountability finds you"
**Demo impact:** Bharath's phone shows calendar notification. More personal/intimate than the public record.
**De-escalation:** Delete the calendar event via API.
**Env vars:**
```
GOOGLE_CALENDAR_CREDENTIALS=path/to/service-account.json
# OR Karen's Gmail OAuth credentials
```

### Level 9: GitHub Open Matters (moved from L8)

**Channel:** `github`
**Integration:** GitHub Contents API (existing, working)
**What fires:** Commits new entry to `data/open-matters.json` in rahilsinghi/portfolio repo. Vercel auto-deploys. Open Matters page at rahilsinghi.com/open-matters updates live.
**Message:** Generated by personality service. Title + body.
**Why moved here:** The permanent public record is a bigger escalation than a calendar invite. This is the penultimate step before the physical letter — it has more weight at Level 9.
**De-escalation:** Removes entry from JSON, commits, Vercel redeploys.

### Level 10: FedEx Formal Letter (ENHANCED)

**Channel:** `fedex`
**Integration:** FedEx Rate API (sandbox) + WeasyPrint PDF generation

**What fires:**
1. **Rate lookup:** Calls FedEx Rate API for overnight delivery quote from Karen's address (10001) to target's address.
   - Success: Displays rate on screen ("$28.40 overnight to {city}, {state}").
   - **Fallback:** If API fails, returns error, or credentials missing → show **$28.40** (hardcoded). Never show "Rate unavailable" — it's a dead moment. $28.40 is a punchline.
2. **Legal letter:** Pre-generated legal-style document from `backend/data/legal_letter_template.md`. Variables filled at runtime (escalation ID, target name, amount, channels used, days outstanding). NOT generated by Claude at demo time.
3. **QR code:** Letter includes a QR code at the bottom linking to the Open Matters entry (from Level 9). When Bharath holds the letter up, audience can scan it. Generated via `qrcode` Python library, embedded as base64 PNG in the HTML template.
4. **PDF generation:** WeasyPrint renders the letter using `openclaw/templates/formal_letter.html`.
5. **Result:** PDF downloadable at `/api/escalation/{id}/letter.pdf`. FedEx rate displayed in the level card.

**FedEx Rate API:**
```
POST https://apis-sandbox.fedex.com/rate/v1/rates/quotes
```
Requires: `FEDEX_API_KEY`, `FEDEX_API_SECRET`, `FEDEX_ACCOUNT_NUMBER`.

**Pre-generated legal letter content** (in `backend/data/legal_letter_template.md`):
- Header: KAREN AUTOMATED CORRESPONDENCE SYSTEMS LLC
- Recipient: {target_name} at {target_address}
- Body: Formal notice of unresolved financial obligation of ${amount}. References {attempt_count} prior communications across {channel_count} channels over {days_elapsed} days. Lists all channels used. States intent to pursue "all available administrative follow-up procedures."
- QR code: Links to `https://rahilsinghi.com/open-matters` (or specific entry URL)
- Closing: "Respectfully and with considerable follow-through, Karen"
- Footer: Ref: KAREN-{escalation_id}, "Karen is always watching. Karen means well."

**Demo impact:** Physical letter held up on stage. QR code is scannable. Rate quote adds realism.
**De-escalation:** "FedEx letter already in transit. Karen's words are on paper now. This cannot be undone."

## New SSE Event Types

```typescript
// Add to existing KarenEvent union:
| { type: "research_step"; step: number; detail: string; pause_ms?: number }
| { type: "research_discovery"; target: string; employer: string; work_email: string; coworker_name: string; coworker_email: string }
| { type: "fedex_rate"; rate: string; service: string; destination: string }
```

The `pause_ms` field on `research_step` tells the frontend how long to delay before showing the NEXT step. Default 400ms, the work email step uses 800ms.

## Startup Pre-check (dev.sh)

Before starting services, `dev.sh` checks:
1. `backend/data/research_cache.json` exists
2. For the primary demo target (bharath), all research fields are filled (no `FILL_BEFORE_DEMO` values)
3. If check fails: print warning in red and refuse to start

```bash
# Research cache validation
python3 -c "
import json, sys
with open('backend/data/research_cache.json') as f:
    cache = json.load(f)
target = cache.get('bharath', {})
unfilled = [k for k, v in target.items() if isinstance(v, str) and 'FILL' in v]
if unfilled:
    print(f'ERROR: research_cache.json has unfilled fields for bharath: {unfilled}')
    sys.exit(1)
" || exit 1
```

## De-escalation Sequence (Updated)

```
1. Remove from Open Matters (GitHub commit)
2. Delete Slack message (via stored ts)
3. Delete Discord post (via stored message_id)
4. Delete Calendar event (via stored event_id)
5. Cancel FedEx shipment (fails if L10 completed — by design)
6. Send apology to target
7. Send apology to CC'd contacts (from L5)
8. Send apology to the apology ("caused confusion")
9. Karen's closing line + next target prompt
```
