# Escalation Ladder v2 — Design Spec

**Goal:** Redesign Karen's 10-level escalation ladder so every level fires a unique channel with real integrations, no stubs, no repeats, and maximum demo impact.

**Replaces:** The original ladder (4x email, 2x SMS, 2 stubs, broken Twitter). This spec is authoritative — it supersedes the level definitions in CLAUDE.md.

---

## The New Ladder

| Level | Channel | What Karen Does | Demo Moment |
|-------|---------|-----------------|-------------|
| 1 | **Email** | Warm first contact from Karen's Gmail | Standard opener |
| 2 | **SMS** | Text message to target's phone | Phone buzzes on mirrored screen |
| 3 | **WhatsApp + Voice Call** | WhatsApp message + Karen calls the phone | Phone rings in the room |
| 4 | **OSINT Research** | Pre-cached "discovery" — Karen finds target's employer, domain, infers work email, identifies coworkers | Research animation plays on screen |
| 5 | **Email CC** | Formal email CC'ing the "discovered coworker" (a circle member) | Payoff from the research step |
| 6 | **Slack** | Posts in #karen-escalations channel | Real Slack message visible to audience |
| 7 | **Discord @everyone** | Posts in demo server | Audience phones ping |
| 8 | **GitHub Open Matters** | Commits to public repo, page goes live | Visible on rahilsinghi.com |
| 9 | **Google Calendar** | Creates event "Discuss $23 Debt" and invites target as attendee | Calendar notification on phone |
| 10 | **FedEx Letter** | Pre-generated legal letter + real FedEx rate quote + PDF | Hold physical letter up on stage |

## Level-by-Level Spec

### Level 1: Email

**Channel:** `email`
**Integration:** Karen's Gmail (karen.follows.up.nyc@gmail.com) via Gmail SMTP or Resend API
**What fires:** Single email to target's email address
**Message:** Generated by personality service. Subject + body. Warm, friendly, one emoji for passive-aggressive.
**Karen's note:** Internal monologue about first contact.
**Audio:** Quip on fire ("Sent. The clock starts now."), commentary during countdown.

### Level 2: SMS

**Channel:** `sms`
**Integration:** Twilio SMS API
**What fires:** Text message to target's phone number
**Message:** Generated by personality service. Body only (no subject for SMS). Short, direct.
**Demo impact:** Bharath's mirrored phone buzzes visibly.
**Audio:** Quip ("He'll see this one."), commentary during countdown.

### Level 3: WhatsApp + Voice Call

**Channel:** `whatsapp`, `voice_call`
**Integration:** Twilio WhatsApp API + Twilio Voice (Polly.Joanna TTS)
**What fires:** WhatsApp message first, then Karen calls the phone and reads a message aloud via TTS.
**Voice call details:** TwiML with Polly.Joanna voice. Message generated by personality service. Fixed closing: "This was a message from Karen Automated Correspondence Systems. Goodbye."
**Demo impact:** Phone rings in the room. Audience hears Karen's Twilio voice if speakerphone.
**Audio:** Quip ("I called. He didn't answer. I noted this."), commentary during countdown.

### Level 4: OSINT Research (NEW)

**Channel:** `research` (new channel type — no external message sent)
**Integration:** Pre-cached research results displayed via SSE events
**What fires:** No message to target. Instead, Karen "investigates" the target:
  1. SSE event: `research_step` — "Scanning {target_name}'s public web presence..."
  2. SSE event: `research_step` — "Found portfolio at rahilsinghi.com" (or target's URL)
  3. SSE event: `research_step` — "Work history: {company} — {role}"
  4. SSE event: `research_step` — "Domain discovered: {domain}. Inferring work email..."
  5. SSE event: `research_step` — "Work email: {first}@{domain}. Confidence: high."
  6. SSE event: `research_step` — "Identifying colleagues for visibility escalation..."
  7. SSE event: `research_discovery` — final payload with discovered contacts

**Pre-cached data structure:**
```python
@dataclass
class ResearchResult:
    target_name: str
    portfolio_url: str
    employer: str
    role: str
    domain: str
    work_email: str
    coworker_name: str       # Actually a circle member
    coworker_email: str      # That circle member's real email
    discovery_steps: list[str]  # The narrative steps above
```

**Storage:** `backend/data/research_cache.json` — one entry per target member. Pre-populated before demo.

**Frontend:** New `ResearchAnimation` component in the timeline. Shows each step appearing with a typewriter effect + terminal-style green text on dark background. Karen's commentary runs alongside ("This is not surveillance. This is due diligence.").

**Demo impact:** The audience watches Karen "think" — most visually interesting level.
**Audio:** Commentary is the star here. Karen narrates the research process.

**De-escalation:** Nothing to reverse (no message was sent).

### Level 5: Email CC

**Channel:** `email_cc`
**Integration:** Karen's Gmail or Resend, with CC field
**What fires:** Formal email to target, CC'ing the "discovered coworker" from Level 4.
**Message:** Generated by personality service with `cc_name` context. Tone shift — now there's an audience.
**CC recipient:** The coworker discovered in Level 4, which is actually a circle member.
**De-escalation:** Apology sent to CC'd contact.

### Level 6: Slack (NEW)

**Channel:** `slack`
**Integration:** Slack Web API (chat.postMessage) via Bot token
**What fires:** Message posted to #karen-escalations channel in Karen's Slack workspace.
**Message:** Generated by personality service. Body only. Can include Slack markdown (bold, links).
**Bot identity:** "Karen" bot with 🦞 avatar.
**Env vars:**
```
SLACK_BOT_TOKEN=xoxb-...
SLACK_CHANNEL_ID=C...
```
**De-escalation:** Delete message via `chat.delete` API.
**Demo impact:** Audience can be invited to the Slack workspace (second QR code alongside Discord).

### Level 7: Discord @everyone

**Channel:** `discord`
**Integration:** Discord Bot API (existing, working)
**What fires:** @everyone message in #karen-text-demo-channel
**Message:** Generated by personality service. Includes @everyone mention.
**Demo impact:** Every audience member's phone pings. This is the signature demo moment.
**De-escalation:** Delete message via Discord API (existing, working).

### Level 8: GitHub Open Matters

**Channel:** `github`
**Integration:** GitHub Contents API (existing, working)
**What fires:** Commits new entry to `data/open-matters.json` in rahilsinghi/portfolio repo. Vercel auto-deploys. Open Matters page at rahilsinghi.com/open-matters updates live.
**Message:** Generated by personality service. Title + body.
**De-escalation:** Removes entry from JSON, commits, Vercel redeploys.

### Level 9: Google Calendar (NEW — real integration)

**Channel:** `calendar`
**Integration:** Google Calendar API via service account OR Karen's Gmail OAuth
**What fires:** Creates a calendar event titled "Discuss $23 Debt — Karen Automated Correspondence Systems" with:
  - Target as invited attendee
  - Event time: 30 minutes from now
  - Description: Generated by personality service
  - 3 reminders: 10min, 5min, 1min before
  - Location: "Wherever accountability finds you"
**Demo impact:** Bharath's phone shows calendar notification.
**De-escalation:** Delete the calendar event via API.
**Env vars:**
```
GOOGLE_CALENDAR_CREDENTIALS=path/to/service-account.json
# OR use Karen's Gmail OAuth token
```

### Level 10: FedEx Formal Letter (ENHANCED)

**Channel:** `fedex`
**Integration:** FedEx Rate API + WeasyPrint PDF generation
**What fires:**
  1. **Rate lookup:** Calls FedEx Rate API for overnight delivery quote from Karen's address to target's address. Displays rate on screen ("$14.72 overnight to {city}, {state}").
  2. **Legal letter:** Pre-generated legal-style document (NOT Claude-generated at runtime — stored in `backend/data/legal_letter_template.md`). Letter references: escalation ID, all previous levels attempted, dollar amount, days outstanding.
  3. **PDF generation:** WeasyPrint renders the letter using the existing formal_letter.html template.
  4. **Result:** PDF downloadable at `/api/escalation/{id}/letter.pdf`. FedEx rate displayed in the level card.

**Pre-generated legal letter content** (stored as template, variables filled at runtime):
- Header: Karen Automated Correspondence Systems LLC
- Recipient: {target_name} at {target_address}
- Body: Formal notice of unresolved obligation. References {attempt_count} prior communications across {channel_count} channels over {days_elapsed} days. States intent to pursue "all available administrative follow-up procedures."
- Closing: "Respectfully and with considerable follow-through, Karen"
- Footer: Escalation reference number, list of all channels used

**FedEx Rate API:**
```
POST https://apis-sandbox.fedex.com/rate/v1/rates/quotes
```
Requires: `FEDEX_API_KEY`, `FEDEX_API_SECRET`, `FEDEX_ACCOUNT_NUMBER`.
If credentials unavailable, skip rate lookup and show "Rate unavailable — Karen will find a way."

**Demo impact:** Physical letter held up on stage. Rate quote adds realism.
**De-escalation:** "FedEx letter already in transit. Karen's words are on paper now. This cannot be undone."

## Setup Guide for User

### Slack Setup (guided, ~10 min)
1. Create Slack workspace (e.g., "Karen HQ")
2. Create #karen-escalations channel
3. Go to api.slack.com/apps → Create New App → From Scratch
4. Name: "Karen", Workspace: Karen HQ
5. OAuth & Permissions → Bot Token Scopes: `chat:write`, `chat:write.public`
6. Install to Workspace → Copy Bot Token
7. Get channel ID: right-click #karen-escalations → Copy Link → extract ID
8. Add to .env: `SLACK_BOT_TOKEN`, `SLACK_CHANNEL_ID`

### Google Calendar Setup (~15 min)
1. Karen's Gmail account: karen.follows.up.nyc@gmail.com
2. GCP Console → Create project "Karen" → Enable Calendar API
3. Create OAuth consent screen (internal) → Create credentials (OAuth client ID)
4. Authorize Karen's Gmail → Get refresh token
5. Karen creates events on her own calendar, invites target as attendee
6. Alternative: Service account approach (share target's calendar with service account)

### FedEx Developer Setup (~10 min)
1. developer.fedex.com → Create account → Create project
2. Get API Key + Secret + Account Number (sandbox)
3. Add to .env: `FEDEX_API_KEY`, `FEDEX_API_SECRET`, `FEDEX_ACCOUNT_NUMBER`

### Circle Data Collection (last step)
Collect from each circle member: email, phone, address. Fill into circle.json.

## De-escalation Sequence (Updated)

```
1. Remove from Open Matters (GitHub commit)
2. Delete Slack message
3. Delete Discord post
4. Delete Calendar event
5. Cancel FedEx shipment (fails if L10 completed — by design)
6. Send apology to target
7. Send apology to CC'd contacts (from L5)
8. Send apology to the apology ("caused confusion")
9. Karen's closing line + next target prompt
```

Removed: LinkedIn recall attempt (gone), Twitter delete (gone).
Added: Slack delete, Calendar delete.
